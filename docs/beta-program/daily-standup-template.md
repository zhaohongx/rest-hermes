# Beta Dogfood Daily Standup

> **时长**：5 分钟（严格） | **频率**：D3-D13 每日
> **数据源**：`python ci/beta-health-check.py` 输出

---

## 议程

| 时间 | 内容 | 负责人 |
|------|------|--------|
| 0:00-1:00 | 昨日调用数 + 反馈数（读 health-check 摘要行） | 轮值 |
| 1:00-2:00 | 关键指标快报（误分类率 / 满意度 / P0） | 轮值 |
| 2:00-3:30 | 异常案例 spotlight（最多 1 个，深度讨论移专项） | 轮值 |
| 3:30-4:30 | 今日 dogfood 重点场景预告 | P3 owner |
| 4:30-5:00 | 阻塞认领 | 全员 |

---

## 红线触发（standup 中立即升级）

- 任意 P0 → 不等散会，立即触发 incident response
- 连续 2 天调用数 < 3 → 会后 15 分钟根因分析
- 误分类率 > 15% → 提前触发 mid-term check
