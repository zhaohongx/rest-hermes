# 语义级矛盾检测启发式规则

> **定位**：补充 formalize v3.0 §2.2 的关键词词对匹配，覆盖**语义张力**（非字面矛盾）场景。
> **触发时机**：S2.2 矛盾约束检测阶段，在词对匹配完成后运行。

---

## 一、与词对匹配的关系

```
输入 → 词对匹配（快速） → 命中？ → 按矛盾追问模板处理
                    ↓ 未命中
              语义启发式（本文件） → 命中？ → 标注 "语义张力" + 追问
                              ↓ 未命中
                           继续形式化
```

词对匹配覆盖**显式矛盾**（如"免费"×"企业级"）。
语义启发式覆盖**隐式张力**（如"简单"但列了 20 个功能）。

---

## 二、启发式规则

### H1：功能数 vs 复杂度宣称

```
触发：用户声称"简单"/"轻量"/"极简" 且 识别到的功能实体/需求 ≥ 5

信号：语义张力——声称的简单度与列出的复杂度不匹配

示例：
"做一个简单的后台管理系统" + 列了 8 个模块
→ ⚠️ 语义张力：宣称简单但功能实体≥5，请确认核心范围

追问模板：
"您提到'简单'，但列出了 ≥5 个功能/模块。
是否接受先做核心 2-3 个模块，其余列入二期？"
```

### H2：时间线 vs 范围

```
触发：时间约束 < 1 周（或"尽快/紧急"）且 识别到的子系统/模块 ≥ 3

信号：时间预算与任务范围不匹配

示例：
"这周内完成" + 涉及前端/后端/数据库/部署
→ ⚠️ 语义张力：时间约束紧但子系统≥3，请确认优先级
```

### H3：成本 vs 质量期望

```
触发：用户提到"免费/零成本/无预算" 且 提到"生产级/高可用/可靠/稳定"

信号：成本约束与质量要求不匹配（词对匹配已覆盖字面矛盾，此规则捕获间接关联）

示例：
"用开源方案就行" + "需要 7×24 运维支持"
→ ⚠️ 语义张力：开源不等于免费运维
```

### H4：通用性 vs 特异性

```
触发：声称"通用/适配所有/任意" 且 需求中描述了高度特化的领域逻辑

信号："万能工具"的宣称与实际领域紧耦合

示例：
"做一个通用的数据导出工具" + 所有示例都是特定 ERP 系统格式
→ ⚠️ 语义张力：宣称通用但需求高度特化
```

### H5：数据量级不一致

```
触发：声称数据量级 且 处理方式与该量级不匹配

信号：
- "海量数据" + 建议用 Excel 处理
- "每天几千条" + 建议用 Hadoop 集群
- "日志分析"不提数据量但建议用 ELK 全量索引

示例：
"分析海量日志" + 无任何数据量参数
→ ⚠️ 语义张力：声称海量但未定义量级，无法评估方案合理性
```

## 三、语义张力追问模板

当命中语义启发式（非词对匹配）时，使用修订版追问模板：

```
检测到潜在约束张力（非字面矛盾）：

> "<原文片段>"

存在以下语义张力：

1. <张力描述> —— 影响：<如果不澄清，设计方案会走向哪个错误方向>
2. 是否接受 <trade-off>？ —— 影响：<可行路径>

请确认后再继续。
```

**与词对矛盾的追问模板区别**：
- 词对矛盾：更强势（"难以同时满足""必须明确优先级"）
- 语义张力：更协商（"潜在张力""请确认"）
- 语义张力不阻塞形式化流程（severity ≤ medium），仅要求标注 ⚠️
