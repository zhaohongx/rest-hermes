# formalize 度量量表

> **定位**：置信度评估、长度约束、追问收敛、失败模式对照——所有"量表类"内容。
> **与 skeleton-card.md 的关系**：skeleton-card 放"规则"，本文件放"刻度"。
> **加载时机**：S8 置信度评估阶段按需加载。

---

## 一、S8 置信度评估量表

| 等级 | 判定条件 | 必须输出 |
|------|---------|---------|
| **高** | 10 维度中 ≥8 ✓，无关键 ⚠️ | 置信度声明 |
| **中** | 10 维度中 ≥6 ✓，1-3 个关键 ⚠️ | 置信度声明 + 关键 ⚠️ 列表 |
| **低** | <6 ✓ 或 ≥4 关键 ⚠️ | 置信度声明 + "降低不确定性的下一步" |

低置信度时，"降低不确定性的下一步"必须是**可执行的具体动作**（如"送样做 XRD 鉴定硅赋存形态"），不可为泛泛的"进一步研究"。

---

## 二、输出长度约束表

| 类型 | 目标长度 | 超限判定 |
|------|---------|---------|
| 跳过 | < 100 字 | >200 字 → 违规 |
| 追问 | < 400 字 | >600 字 → 违规 |
| L1 微型 | 300-600 字 | — |
| L2 标准 | 800-1800 字 | — |
| L2 + STEM | 1200-2500 字 | — |
| L3 扩展 | 1500-3500 字 | — |

**反模式**：简单问题写 5000 字 / 复杂问题写 500 字 / 章节缺失。

---

## 三、追问数收敛表

| 输入复杂度 | 追问数硬上限 |
|-----------|-------------|
| 简单（单实体单需求） | ≤ 1 个 |
| 中等（2-5 实体） | ≤ 2 个 |
| 复杂（>5 实体或系统级） | ≤ 3 个 |

**合并规则**：若初步识别出上限数以上问题，必须检查能否合并。禁止通过拆分单一关注点凑数。

---

## 四、失败模式对照表

| 失败模式 | 表现 | 修正 |
|---------|------|------|
| 该跳过没跳 | 给原子修改输出完整规格 | 强化 S0 |
| 该追问没追 | 对"做个智能系统"直接形式化 | 强化 S1 |
| 追问过多 | >3 个问题 | 收敛到 ≤3 |
| 简单问题过度 | L1 用 L3 模板 | S2 分级 |
| 模板飘忽 | 章节标题随意改 | S7 自检 |
| STEM 缺公式 | 物理题没列方程 | B1-STEM |
| 假装确认 | 未验证 API 没标 ⚠️ | S6 |
| 模糊词泛滥 | 充斥"大概/应该" | S7 自检 |
| 置信度缺失 | 末尾无置信度 | S7 自检 |
| 推荐缺冷启动 | 推荐系统未提 | 失败模式 1 |
