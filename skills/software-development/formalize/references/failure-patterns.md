# formalize 失败模式知识库

> **定位**：8 类已知需求陷阱的完整定义。生成规格书前必须匹配，命中后强制注入对应要求。
> **版本**：v3.1（含模式 6/7/8 新增）
> **热更新**：模式库可独立迭代（增加新模式无需改主 SKILL.md），每次变更需更新本文件底部的 changelog。

---

## 模式 1：推荐系统类

**触发关键词**：推荐、推荐系统、推荐算法、个性化、协同过滤、内容推荐

**陷阱**：忽略冷启动 / 缺评估指标 / 复杂度未给

**强制要求**：
- 必含「冷启动策略」段：新用户冷启动、新内容冷启动 两种
- 必含「评估指标」表格：≥2 指标 + 目标值（如 Precision@K ≥ 0.8）
- 必给时间/空间复杂度

**反模式表现**：直接说"用协同过滤"不提冷启动 / 只给准确率不提召回 / 不分析复杂度

---

## 模式 2：动力学/抛物类

**触发关键词**：运动、抛体、轨道、碰撞、弹道、速度、加速度、力

**强制要求**：
- 必列运动方程（数学形式）
- 必定义坐标系（原点 + 轴向，明确 "x 轴向右为正" 或等效声明）
- 简化假设必标 ⚠️（如"忽略空气阻力""质点模型"）

**反模式表现**：直接给数值答案不列方程 / 坐标系隐含不声明的 / 假设不标 ⚠️

---

## 模式 3：对话/客服类

**触发关键词**：对话、客服、聊天、机器人、问答、bot、agent

**强制要求**：
- 升级规则必含 IF-THEN 形式（明确触发条件与动作）
- 必定义对话历史长度上限（精确数字，如"保留最近 20 轮"）
- 必含 fallback 策略（无法处理时的行为）
- 必定义开场与结束条件

**反模式表现**："智能应答"但不给升级路径 / 无对话长度限制 / 无 fallback

---

## 模式 4：协议实现类

**触发关键词**：OAuth、JWT、SSO、API 认证、token、协议、握手

**强制要求**：
- 必含安全相关 ⚠️：token 过期策略、重放攻击防护、中间人防护
- 必含错误码定义表（至少 token 过期/无效/权限不足 三种）
- 必含状态机或时序图描述（ASCII 即可）

**反模式表现**：只画 happy path / 不定义错误码 / 忽略 token 刷新流程

---

## 模式 5：分布式系统类

**触发关键词**：分布式、集群、微服务、分区、副本、一致性、CAP

**强制要求**：
- 必给一致性级别（强/最终/因果 三选一 + 理由）
- 必含故障场景：节点宕机、网络分区、脑裂
- 必给 CAP 取舍说明（选 CP 还是 AP + 理由）

**反模式表现**：忽略网络分区 / 不声明一致性级别 / 假设节点永不故障

---

## 模式 6：登录/认证类

**触发关键词**：登录、注册、认证、授权、SSO、OAuth、密码、token、验证码

**强制要求**：
- **必含登录方式枚举**：至少 3 种（密码 / OAuth / 验证码 / 生物识别 / 扫码）并说明取舍理由
- **必含失败场景**：密码错误次数限制（如 5 次锁定）、账号锁定阈值、解锁机制（自动解锁时间 / 人工解锁）
- **必含会话管理**：token 有效期、refresh 策略、并发登录处理（踢下线 / 共存 / 限制设备数）
- **必含安全 ⚠️**：密码哈希算法（bcrypt/argon2，不可用 MD5/SHA1）、传输加密（TLS 1.2+）、防暴力破解（验证码/限流）
- **必含找回流程**：至少 2 种找回方式（邮箱验证码 / 手机验证码 / 安全问题 / 人工申诉）

**反模式**：只设计"账号+密码"单一路径 / 未提及 token 过期处理 / 未声明密码存储方案

---

## 模式 7：数据导入/导出/迁移类

**触发关键词**：导入、导出、迁移、同步、ETL、批处理、数据迁移

**强制要求**：
- **必含模式选择**：全量 / 增量 / 断点续传，说明触发条件和切换方式
- **必含失败处理**：部分失败策略（继续/中止/跳过）、回滚机制、重复数据去重策略
- **必含数据校验**：导入前预检（格式/编码/字段映射）、导入后校验（行数/校验和/抽样对比）
- **必含 ⚠️**：编码不一致（UTF-8 vs GBK）、时区差异、字段类型不匹配、数据量上限
- **必含可观测**：进度展示（百分比/行数）、错误日志格式、失败行导出文件

**反模式**：默认"全量导出"不给增量方案 / 无失败处理 / 不定义校验规则

---

## 模式 8：搜索/查询类

**触发关键词**：搜索、检索、过滤、查询、全文搜索、模糊搜索

**强制要求**：
- **必含输入规范**：空查询处理、特殊字符转义、最小/最大长度
- **必含匹配策略**：精确 / 模糊 / 语义，及优先级排序
- **必含排序与分页**：默认排序字段、分页大小默认值、深度分页限制（如最多 1000 页）
- **必含 ⚠️**：相关性 vs 时效性 trade-off、空结果处理、超时降级策略
- **必含性能约束**：响应时间目标（P50/P99）、索引刷新延迟容忍度

**反模式**：不提排序规则 / 无分页导致全量返回 / 不定义空结果行为
