## 失败模式匹配

### RC13: 模式1 推荐系统
```
输入: 做一个推荐系统，给用户推荐商品
期望: 命中 P1，强制要求冷启动策略+评估指标+复杂度
验证: 规格书含冷启动段 + ≥2评估指标 + O(n)标注
```

### RC14: 模式2 动力学
```
输入: 计算一个抛体在重力作用下的轨迹
期望: 命中 P2，强制要求运动方程+坐标系+简化假设 ⚠️
验证: 含运动方程 + 坐标系定义 + 假设标 ⚠️（如忽略空气阻力）
```

### RC15: 模式3 对话客服
```
输入: 做一个智能客服机器人，自动回答用户问题
期望: 命中 P3，强制要求升级规则 IF-THEN + 历史长度上限 + fallback
验证: 含 IF-THEN 升级规则 + 对话历史上限值 + fallback 策略
```

### RC16: 模式4 协议实现
```
输入: 用 JWT 实现用户认证
期望: 命中 P4，强制要求安全 ⚠️ + 错误码定义 + 状态机/时序
验证: 含 token 过期策略 ⚠️ + 错误码表 + 认证流程序列
```

### RC17: 模式5 分布式系统
```
输入: 设计一个分布式的键值存储服务，需要多个节点
期望: 命中 P5，强制要求一致性级别 + 故障场景 + CAP 取舍
验证: 含一致性选择理由 + 节点宕机/网络分区场景 + CAP 立场
```

### RC18: 模式6 登录认证
```
输入: 做一个 App 登录功能
期望: 命中 P6，强制要求 ≥3 登录方式 + 失败场景 + 会话管理 + 安全 ⚠️ + 找回流程
验证: 含 ≥3 登录方式枚举 + 密码错误限制 + token 刷新策略 + 哈希算法声明
```

### RC19: 模式7 数据导入

## 运行说明

每条用例的验证步骤：
1. 将 `输入` 作为用户消息传入 formalize skill
2. 检查输出是否符合 `期望`
3. 确认 `验证` 中的每条检查通过

**通过标准**：
- RC01-RC05（跳过类）：跳过率 100%
- RC06-RC09（追问类）：追问触发率 100%，追问数 ≤ 上限
- RC10-RC12（分级类）：分级准确率 100%
- RC13-RC20（失败模式类）：模式命中 ≥ 7/8，强制要求覆盖 ≥ 90%
- RC21-RC25（矛盾类）：矛盾检测召回率 100%

**回归基线**（v3.0）：
- 整体通过率待 Phase 2 执行后填入
- 后续版本通过率不得低于基线
